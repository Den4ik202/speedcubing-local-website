# 🏆 Спидкубинг Красноярск — Система управления турнирами

Локальное веб-приложение для организаторов соревнований по спидкубингу. Позволяет управлять турнирами, регистрировать участников и быстро вводить результаты попыток с автоматическим расчетом по правилам WCA.

Система разработана так, чтобы её можно было запустить на главном компьютере (сервере), а результаты вводить с любого устройства (например, со смартфонов судей), подключенного к той же Wi-Fi сети.

---

## ✨ Ключевые возможности

* **Управление соревнованиями:** Создание турниров, добавление дисциплин и регистрация участников.
* **Умный ввод результатов:** * Ввод формата "справа налево" (как в WCA Live / csTimer).
    * Горячие клавиши `D` (DNF) и `S` (DNS).
    * Автоматический переход между попытками по нажатию `Enter`.
* **Калькулятор WCA:** Автоматический расчет **SINGLE** (лучшая попытка) и **AVG5** (среднее из 5 с отбрасыванием лучшей и худшей попытки).
* **Формирование таблиц лидеров:** Мгновенное обновление рейтингов (сортировка по AVG5, затем по SINGLE).
* **Excel как база данных:** Все данные надежно сохраняются в файл `speedcubing_db.xlsx`. Его всегда можно открыть, проверить вручную или перенести на другой ПК.

## 🚀 Запуск для разработки (Dev Mode)

Если вы хотите запустить исходный код или внести изменения:

1. Убедитесь, что установлен Python 3.8 или выше.
2. Склонируйте репозиторий или скачайте архив с кодом.
3. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv .venv
   # Для Windows:
   .venv\Scripts\activate
   # Для Mac/Linux:
   source .venv/bin/activate
   
## 📦 Сборка и упаковка проекта в .exe

1. Откройте терминал в папке с проектом и убедитесь, что ваше виртуальное окружение активировано.
2. Убедитесь, что установлена библиотека pyinstaller (она есть в requirements.txt).
3. Выполните команду сборки. Она зашьет все HTML-шаблоны и статические файлы прямо внутрь .exe:
```bash
pyinstaller --onefile --add-data "templates;templates" --add-data "static;static" app.py
```
4. Дождитесь окончания процесса сборки (около 1-2 минут).
5. После завершения в папке вашего проекта появится новая папка dist. Внутри неё будет находиться готовый файл app.exe.

## Подготовка файлов для конечного пользователя
Создайте пустую папку (например, Speedcubing_App) и положите в неё:
1. Файл app.exe (из папки dist).
2. Файл базы данных speedcubing_db.xlsx (пустой или с уже настроенными дисциплинами).

## Инструкция для конечного пользователя (Организатора)
1. Скачайте и распакуйте полученный ZIP-архив в любое удобное место на компьютере.
2. Дважды кликните по файлу app.exe.
3. Откроется черное окно консоли (сервер) — не закрывайте его, пока идет соревнование.
4. В этом окне появится красивый текст со ссылками. Выделите ссылку http://127.0.0.1:5000 (или ссылку с вашим IP), скопируйте её и вставьте в браузер.
5. Готово! Система работает. Чтобы закончить работу, просто закройте черное окно консоли. Все результаты автоматически сохранятся в лежащий рядом файл speedcubing_db.xlsx.
